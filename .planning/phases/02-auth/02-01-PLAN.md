# Plan 02-01: Authentication & User Management

## Goal
Implement a robust JWT-based authentication system with ASP.NET Identity and refresh token support.

## Tasks
- [ ] Task 1: Identity & User Model
  - Create `ApplicationUser` in `Domain`.
  - Update `ApplicationDbContext` to use `ApplicationUser`.
  - Configure Identity services in `Infrastructure`.
- [ ] Task 2: JWT & Token Services
  - Define `JwtSettings` in `Application`.
  - Implement `ITokenService` in `Infrastructure` to handle JWT generation and refresh token logic.
  - Update `appsettings.json` with JWT key provided by user.
- [ ] Task 3: Auth Features (CQRS)
  - Implement `RegisterUser` Command & Handler.
  - Implement `LoginUser` Query & Handler.
  - Implement `RefreshToken` Command & Handler.
- [ ] Task 4: Auth Controllers & Middleware
  - Create `AuthController`.
  - Configure JWT Authentication middleware in `API`.
  - Add Authorization policies.
- [ ] Task 5: Database Migrations
  - Add and apply initial migration for Identity tables.
- [ ] Task 6: Frontend Auth Integration
  - Create `useAuth` hook.
  - Implement Login/Register pages.
  - Add Axios/Fetch interceptor for token refresh.
  - Protect routes using a `ProtectedRoute` component.

## Verification
- [ ] User can register and login via API.
- [ ] JWT token is valid and contains user claims.
- [ ] Refresh token successfully generates a new JWT.
- [ ] Frontend routes are inaccessible without a valid token.
