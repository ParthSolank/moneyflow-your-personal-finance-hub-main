# Plan 03-01: Core Financial Modules

## Goal
Implement robust CRUD functionality for Accounts and Transactions with user scoping and real-time balance tracking.

## Tasks
- [ ] Task 1: Domain Entities & Configuration
  - Define `Account` and `Transaction` entities in `Domain`.
  - Configure relationships and user-scoping in `Infrastructure`.
- [ ] Task 2: Account Features (CQRS)
  - Implement CRUD commands/queries for Accounts.
  - Logic to calculate "Current Balance" based on transactions.
- [ ] Task 3: Transaction Features (CQRS)
  - Implement CRUD commands/queries for Transactions.
  - Add filtering by Date, Category, Account, and Type.
- [ ] Task 4: API Controllers
  - Create `AccountsController` and `TransactionsController`.
  - Add `[Authorize]` attributes.
- [ ] Task 5: Database Migrations
  - Add and apply migration for Accounts and Transactions.
- [ ] Task 6: Frontend Integration
  - Replace mock data in `Ledgers.tsx` and `Transactions.tsx`.
  - Implement React Query hooks for fetching and mutations.
  - Add optimistic updates for transaction creation.

## Verification
- [ ] User can create multiple accounts.
- [ ] Transactions correctly affect account balance.
- [ ] Filters in Transactions page work as expected.
- [ ] Data is strictly user-scoped (verified via multiple users).
