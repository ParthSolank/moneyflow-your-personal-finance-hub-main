# Plan 04-01: Budgeting & Categorization

## Goal
Implement a budgeting engine that allows users to set and track spending limits by category.

## Tasks
- [ ] Task 1: Budget Entity & Configuration
  - Create `Budget` entity in `Domain`.
  - Add `Budgets` DbSet to `ApplicationDbContext`.
- [ ] Task 2: Budget Features (CQRS)
  - Implement `GetBudgets` query (returns limit and actual spending).
  - Implement `UpsertBudget` command (create or update limit).
  - Logic to aggregate actual spending for a given month/category.
- [ ] Task 3: API Controllers
  - Create `BudgetsController`.
- [ ] Task 4: Database Migrations
  - Add and apply migration for Budgets.
- [ ] Task 5: Frontend Integration
  - Create `budgetService`.
  - Create `Budgeting` page (or update existing if any).
  - Integrate progress bars showing "Used vs Available".

## Verification
- [ ] User can set a budget for a category (e.g. "Food").
- [ ] The actual spending updates automatically as transactions are added.
- [ ] Budget progress accurately reflects the percentage of limit used.
